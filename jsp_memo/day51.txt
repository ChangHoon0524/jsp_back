+) multipart/form-data에서는 getParameter()가 안되는가?
	request.getParameter()는 내부적으로 x-www-form-urlencoded 방식만 파싱되도록 설계됨(title=hello&content=text)(key=value&key=value 방식)
	
	multipart는 
	----------boundary
	Content-Disposition:form-data
	서버가 자동으로 파싱을 못하고 결과는 null
	
=> 일반적인 폼 요청은 key=value&key=value 형식으로 요청되므로, servlet 컨테이너가 자동으로 파싱하여 getParameter()로 값을 꺼내 사용할 수 있다.
	multipart/form-data는 파일 데이터가 포함되기 위해 boundary 기반 복합 데이터 구조로 전송되므로 서블릿이 기본적으로 파싱할 수 없다.
	결과적으로 request.getParameter()를 사용하면 값이 null이 되고 이 문제를 해결하기 위해 Multipart 전용 파서 라이브러리를 사용해야한다.
	우리가 사용한 라이브러리는 MultipartRequest(cos.jar)를 사용했다

+) MultipartRequest를 이용한 데이터 파싱
 MultipartRequest multipartRequest 
= new MultipartRequest(request, UPLOAD_PATH, FILE_SIZE, "utf-8", new DefaultRenamePolicy()); 
	MultipartRequest 생성 순간 => 업로드 + 파싱 동시 수행
	request : HTTP 요청 객체
	UPLOAD_PATH : 파일 저장 경로
	FILE_SIZE : 파일 최대 크기(5MB)
	"utf-8" : 파일명 인코딩 방식
	new DefaultFileRenamePolicy() : 파일명 중복 시 자동으로 이를 변경해주는 정책
	
	요청 글쓰기 버튼 클릭 -> 글작성 페이지 응답 
[사용자] -> [form] -> [Controller] -> [DAO(DB)] -> [파일 저장] -> [응답]
