복습 -> 전체 흐름 -> 프로젝트 최종 예제1

0. 복습
1) MyBatis 동작 구조
	SqlSessionFactoryBuilder	설정 파일을 읽어 Factory 생성
	↓
	config.xml			DB연결, Mapper 등록, 환경설정
	↓
	SqlSessionFactory		SqlSession을 생성하는 공장
	↓
	SqlSession			Sql 실행 객체
	↓
	Mapper				SQL 실행 인터페이스 (DAO클래스로 생각)
	↓
	Mapper.xml			실제 SQL 쿼리문 작성
	↓
	DB				데이터 저장

2) MVC 구조
	Model		데이터 처리 영역
		 DB 접근, SQL 실행, 데이터 처리
	View		화면 출력(사용자 화면)
		데이터 출력, 입력 폼 제공
	Controller	요청 처리
		요청 받기, 기능 판단, Model 호출, View 연결

3) Model2 구조
	JSP와 Servlet 역할을 분리한 구조
	
- 요청 흐름
	사용자 -> JSP -> web.xml -> FrontController -> 각 기능 Controller -> DAO -> Mapper(SQL) -> DB -> 결과 반환 -> JSP(View)

4) Execute 인터페이스
	모든 Controller가 따라야하는 규칙
	Controller는 반드시 execute() 메소드를 가져야한다
	요청 처리, 결과 반환하는 역할

5) Result 클래스
	페이지 이동 정보를 저장
	path		이동할 페이지
	isRedirect	redirect 여부

6) 세션과 쿠키
	쿠키(Cookie) : 브라우저에 저장되는 데이터
		Server -> Cookie 생성
		Browser -> Cookie 저장
		Browser -> 요청 시 자동 전송
	세션(Session) : 서버가 사용자 정보를 저장하는 공간
		Browser -> JSESSIONID(쿠키) -> Server Session -> 사용자 데이터
