0. 복습
1) HttpServletRequest 관련 메소드

- getParameter(String name)
	단일값 받을 때 사용
	text, radio, select(one)등

- getParameterValues(String name)
	여러 값 받을 때 사용
	checkbox 전용

- setAttribute(String key, Object value)
	forward 방식에서 데이터 전달

- getAttribute(String key)
	jsp에서 request영역 데이터 꺼낼 때

2) RequestDispatcher(forward 관련)
- getRequestDispatcher(String path)
	이동할 페이지를 지정

- forward(request, response)
	서버 내부 이동, request 유지, 주소창 안 바뀜

3) HttpServletResponse 관련
- sendRedirect(String location)
	클라이언트 재요청, request 유지 안됨, 주소창 변경

4) 인코딩 관련
- setCharacterEncoding(String encoding)
	POST 방식에서 한글 깨짐 방지

- URLEncoder.encode(name, "UTF-8")
	redirect 시 한글 깨짐 방지

5) forward와 redirect
- forward
	서버가 받은 요청을 그대로 유지한 채, 서버 내부에서 다른 자원으로 넘기는 방식
	클라이언트(브라우저)는 이동 사실을 모른다

	브라우저 -> 서버 요청
	서버 내부에서 다른 자원으로 전달
	최종 결괌나 브라우저에게 응답

	request.setAttribute() 값 그대로 사용하므로 request 데이터 유지

- redirect
	서버가 브라우저에게 이 주소로 다시 요청하려고 응답하는 방식
	브라우저가 새로 요청을 다시 보낸다
	브라우저 -> 서버 요청
	서버 -> 다른 주소로 가야돼 응답
	브라우저 -> 새 주소로 다시 요청
	요청이 2번 발생

+) 언제 무엇을 써야하는지
	단순 화면 이동 + 데이터 전달 -> forward
	데이터 처리 완료 후 새로운 화면 이동 -> redirect
	새로고침 중복 방지 -> redirect

6) get과 post
	http에서 서버로 데이터를 보내는 방식

- get 방식
	서버에 데이터를 요청할 때 사용하는 방식
	주로 조회(검색, 목록 보기) 용도
	데이터가 url 뒤에 붙어 전송

- post 방식
	서버에 데이터를 제출할 때 사용하는 방식
	주로 저장, 수정, 삭제 같은 작업
	데이터가 HTTP Body(본문)에 담겨 전송 -> url에 보이지 않음

1. JSP 태그
	HTML 안에 Java 코드를 넣기 위해 만들어진 기술
	java코드가 길어지면 유지보수가 어려워졌고 jstl과 el이 등장

1) 지시자 태그(Directive Tag)
	<%@ ... %>
	페이지 설정을 담당

- page 지시자
	<%@ page language="java" contentType="text/html; charset=UTF-8" %>
	JSP 페이지 속성 설정
	인코딩, 에러페이지 지정 등

- include 지시자
	<%@ include file="header.jsp" %>
	다른 jsp 파일을 포함
	컴파일 시점에 포함(소스 통합)

- taglib 지시자
	<%@ taglib url="http://java.sun.com/jsp/jstl/core" prefix="c" %>
	JSTL 사용 선언

2) 선언 태그
	<%! 변수 또는 메소드 선언 %>
	클래스 멤버로 선언
	<%! int count=0; %>

3) 표현식 태그
	<%= 값 %>
	값을 화면에 출력
	현재는 EL로 대체되어 사용됨

4) 스크립틀릿 태그
	<% 자바코드 %>
	거의 사용되지 않음(MVC 분리 되었기 때문)

5) JSP 주석
	<%-- -->
	JSP 코드 주석 시 반드시 사용

6) 액션 태그
	<jsp:forward>
	<jsp:include>
	요즘은 JSTL/EL 때문에 사용빈도가 하락

2. JSTL, EL
1) EL(Expression Language)
	JSP에서 데이터를 출력하기 위한 표현식 언어

	${} 사용
	request, session, application 영역 데이터를 쉽게 꺼낼 수 있음
	자바 코드<% %>없이 값 출력 가능
	<%= request.getAttribute("userName") %>
	${userName}
	
2) JSTL(JSP Standard Tag Library)
	JSP에서 자바 로직(조건문, 반복문 등)을 태그로 처리하는 라이브러리
	외부 라이브러리 필요
	- jstl.jar, standard.jar

3. JSTL 
1) 사용 선언
	<%@ taglib uri="http://java.sun.com/jsp/jstl/core" prefix="c" %>
2) 주요 태그
- 변수 선언
	<c:set var="변수명" value="실제값" />
- 출력
	<c:out value="${변수명}" />
- 조건문
	<c:if test="${age > 20}">
		성인입니다
	</c:if>
- 다중조건
	<c:choose>
	 <c:when test="${score >= 90}">
		A
	 </c:when>
	 <c:otherwise>
		B
	 </c:otherwise>
	</c:choose>

- 반복문
	for(String n : list){}

   	<c:forEach var="i" begin="1" end="10">
  	   ${i}
 	</c:forEach>
		
	<c:forEach var="n" items="${list}">
	      ${n}
	</c:forEach>

4. 스코프(Scope)
	이 값이 어디까지 살아있고, 어디서 사용할 수 있는지를 정하는 범위

1) page 스코프
	현재 jsp 페이지 안에서만 사용 가능
	pageContext.setAttribute("name", "홍길동");
		${name}
	페이지에서만 유효
	다른 페이지로 이동하면 사라짐
	
2) request 스코프
	하나의 요청(request)이 끝날 때까지 유지됨
	ex) request.setAttribute("name", "철수");
		Servlet - JSP 데이터 전달할 때 사용
		forward 하면 유지, redirect하면 사라짐

3) session 스코프
	브라우저가 종료되기 전까지 유지
	ex) session.setAttribute("user", user);
		로그인 정보 저장
		여러 페이지에서 계속 사용 가능

4) application 스코프
	서버가 종료되기 전까지 유지
	모든 사용자가 공유
	ex) application.setAttribute("count", 100);
		방문자 수
		공통 설정 값

5. EL 연산자
1) 산술연산자
	+	더하기
	-	빼기
	*	곱하기
	/	나누기
	%	나머지
	mod	나머지

2) 비교(관계) 연산자
	>	gt
	<	lt
	>=	ge
	<=	le
	==	eq
	!=	ne
	HTML과 충돌 방지를 위해 문자형으로 자주 사용

3) 기타연산자
	empty	값이 다음 중 하나면 true(null, "", 길이가 0인 배열, 길이가 0인 컬렉션)
		null 비교 시 empty로 사용하여 비교
		${name == null} x
		${empty name} o

+) 자바의 메소드를 el에서 바로 사용 불가
fn 라이브러리를 선언 후 사용해야한다
	<%@ taglib uri="http://java.sun.com/jsp.jstl/functions" prefix="fn" %>

1) 문자열 길이
	${fn:length(name)}
	문자열 길이, 배열길이, list 크기

2) 대소문자 변환
	fn:toUpperCase()
	fn:toLowerCase()

3) 문자열 포함 여부
	fn:contains(name, "kim")	// 대소문자 구분하여 포함여부 체크
	fn:containsIgnoreCase(name, "KIM")	// 대소문자 구분 x

4) 문자열 시작 / 끝
	fn:startsWith(name,'김') // 시작
	fn:endsWith(name, '수') // 끝

5) 문자열 자르기
	fn:substring(name, 시작idx, 종료idx)
	시작 idx 포함, 종료 idx 미포함

6) 문자열 치환
	fn:replace(name, '기존값', '변경할값')

7) 공백 제거
	fn:trim()
	앞 뒤 공백 제거

8) 문자열 분리
	fn:split()

9) 배열을 문자열로 합치기
	fn:join()

+) EL과 JAVA 차이	
			JAVA		EL
	"10" + 5	"105"		15
	null + 5	에러		5
	null 비교		NPE 가능		안전
	메소드 호출		가능		불가(fn)

	EL은 자동 형변환한다
	null은 숫자 연산 시 0 취급
	문자열 + 숫자 -> 숫자 변환 후 계산 
	empty는 null과 다르다
	eq 사용 권장
