0. 복습
1) 웹서버와 WAS
	브라우저 -> 웹서버 -> WAS(서블릿 컨테이너) -> 응답

2) 웹서버(Web Server)
	대표 : Apache HTTP Server
	HTTP 요청 수신
	정적 파일 응답
	동적 요청은 WAS로 전달

- 정적 컨텐츠 : html, css, js, 이미지 등
- 동작 흐름 
	클라이언트가 /index.html 요청 -> 서버가 파일 검색 -> 그대로 응답

3) WAS 서버
	대표적인 예 : Apache Tomcat
	
	정적 + 동적 처리 가능
	서블릿 실행
	DB 접근
	비즈니스 로직 처리

- 동적 컨텐츠 : 게시판 목록, 로그인 후 마이 페이지, 주문 내역 등

4) 아파치 톰캣
   WAS, Servlet Container 포함, 웹서버기능 포함
   정적 요청 -> 직접 처리
   동적 요청 -> 컨테이너로 전달 -> 처리후 응답

5) 서블릿(Servlet)
	자바 기반 서버 프로그래밍 기술
	동적 페이지 생성	
	HttpServlet 상속

- 서블릿 처리 흐름
	클라이언트요청(GET/POST)
	웹 서버 -> 컨테이너 전달
	서블릿 객체 생성(최초 1회)
	service() 호출
	doGet() / doPost() 실행
	HTML 생성 후 응답

6) JSP
	Servlet에서 HTML 작성이 너무 불편해서 등장
	HTML 안에 Java코드 작성
	JSP 실행 시 Servlet으로 변환(즉, JSP도 결국 서블릿)

7) ContextPath와 서블릿 경로
	ontextPath : 웹 어플리케이션의 루트 경로(프로젝트 단위)
      서버에 여러 프로젝트가 올라가 있을 때 각 프로젝트를 구분하는 이름
      프로젝트 단위로 하나
      보통 프로젝트명과 동일하거나 /
      request.getContextPath()로 가져올 수 있음
      서버의 최상위 경로가 아니라 해당 웹 어플리케이션의 최상위 경로

   Servlet 경로(Servlet Mapping Path) : 특정 서블릿과 연결된 URL 경로(서블릿 단위)
      사용자가 이 주소를 입력하면 해당 서블릿이 실행된다

      - 어노테이션 방식
      @WebServlet("/output")

      컨텍스트 패스/output 

      - web.xml 방식
      <servlet-mapping>
         <servlet-name>MyServlet</servlet-name>
         <url-pattern>/output</url-pattern>
      </servlet-mapping>

   서버주소 + ContextPath + Servlet 경로
   http://localhost:8888/day01/output


1. web.xml
	.xml(Extensible Markup Language)
		데이터를 구조화해서 저장하고 전달하기 위한 마크업언어

- xml 문법 규칙
	모든 태그는 닫혀야함
	루트 요소는 하나만 존재
	태그는 중첩 구조를 명확히 지켜야함
	속성값은 반드시 따옴표 사용

- 사용예시
	웹서비스 데이터 교환(ex : SOAP) 
	설정파일(ex : Spring maven)
	문서포맷(ex : Office Open XML)
	RSS 피드

+) html과 xml 차이
		html			xml
	목적	웹 페이지 표시		데이터 저장/전달
	태그	미리 정의			사용자가 정의
	문법	비교적 느슨			엄격
	
2) web.xml
	Deployment Descriptor(배포 설명자) : 웹 어플리케이션의 설정 파일
	위치 : WEB-INF/web.xml
	WEB-INF 폴더 안에 있기 떄문에 브라우저에서 직접 접근 불가, 서버만 읽는다.

3) 필요한 이유
	과거에는 어노테이션이 없었기 때문에 서블릿과 URL을 반드시 이 파일 안에서 연결해야했다.
	현재 필요한 경우
	- 어노테이션을 사용하지 않을 때
	- 초기 설정 필요할 때
	- 필터/리스너 등록할 때
	- 프로젝트 전체 설정

4) 기본구조
<?xml version="1.0" encoding="UTF-8"?>

루트 태그, Java EE 웹 어플리케이션 설정의 시작점, version=4.0 은 Servlet 4.0 스펙기준
<web-app xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xmlns="http://xmlns.jcp.org/xml/ns/javaee" xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee http://xmlns.jcp.org/xml/ns/javaee/web-app_4_0.xsd" version="4.0"> // 여기까지 루트 태그
- 네임스페이스 부분
	xmlns			기본 네임페이스 정의
	xmlns:xsi		XMLSchema 관련 속성 사용 선언
	xsi:schemaLocation	해당 XML이 어떤 XSD 스키마를 따르는지 명사
	web-app_4.0.xsd Selvr 4.0 규격에 맞는지 검증
	=> 이 파일이 Servlet 4.0 표준에 맞게 작성되었는지 검사하겠다는 의미

어플리케이션 이름, 관리콘솔 등에서 표시될 이름

현재 web.xml 기준
	MyServlet		/day02/MyServlet
	InputServlet		/day02/input
	OutputServlet		/day02/output

/input 접속
	이름/나이/아이디/비밀번호 입력 => name 속성, action="output", method
	submit 버튼

/output 요청
	result.jsp 페이지로 이동

forward 방식으로 이동
	request.getRequestDispatcher("/result.jsp")
	request : 요청 객체
		브라우저가 서버로 보낸 요청 정보 전체
		파라미터, 헤더, 세션정보, request에 저장한 데이터

	=> 어디로 이동할 지를 지정하는 것

- getRequestDispatcher("/result.jsp")
	서버 내부에서 /result.jsp로 이동 준비
	/	현재 웹 어플리케이션의 루트 기준

- forward(request, response)
	request : 지금까지 사용하던 그 request 객체를 그대로 넘겨준다
		InputServlet에서 받은 데이터,
		OutputServlet에서 setAttribute한 값,
		getParameter로 받은 값
	전부 그대로 유지한 채 jsp로 전달

forward 전체 흐름
   브라우저 전송버튼 클릭시 -> /output 요청 (web.xml에서 /output 경로가 있는 지 확인 후)
   OutputServlet 실행
   request에 데이터 저장
   forward(request, response)
   result.jsp 실행
   request 데이터 사용 가능

2. GET과 POST
1) GET
	데이터를 URL에 붙여서 전송

2) POST
	데이터를 요청 본문(body)에 담아서 전송
	URL에는 데이터가 보이지 않는다

3) 서블릿에서의 차이
- GET 요청
	protected void doGet(HttpServletRequest request, HttpServletResponse response)

- POST 요청
	protected void doPost(HttpServletRequest request, HttpServletResponse response)

4) 폼에서 사용하는 방법
- GET 방식
	<form method="get" action="서블릿경로">
- POST 방식
	<form method="post" action="서블릿경로">

5) 중요한 차이
- URL 노출 여부
	GET : 비밀번호 노출됨	ex) /output?name=kim&pw=1234
	POST : URL에 안보임	ex) /output

- 한글 처리 여부
	POST는 반드시 필요		request.setCharacterEncoding("UTF-8");
	GET은 서버 설정에 따라 다름

- 내부 동작 차이
	GET : Header + QueryString
	POST : Header + Body

+) 멱등성(idempotent)
	GET : 여러번 요청해도 결과 동일
	POST : 여러번 요청하면 데이터 중복 가능

★ 요청 -> 서블릿경로(web.xml) -> 서블릿실행 -> get/post 실행 -> 응답
