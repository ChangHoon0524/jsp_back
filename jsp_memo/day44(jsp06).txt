MyBatis Framework

1. MyBatis Framework
1) 프레임워크
	이미 정해진 구조와 흐름이 있고 그 틀 안에서 코드를 작성하는 것

   라이브러리
	내가 필요할 때 가져다 쓰는 도구 모음

2) MyBatis Framework
	자바에서 관계형 데이터베이스(RDBMS)를 쉽게 다루도록 도와주는 프레임워크
	
- JDBC
	드라이버 로드
	커넥션 연결
	PreparedStatement 생성
	SQL 작성
	ResultSet 처리
	객체에 데이터 담기
	자원 해제

3) MyBatis 장점
- JDBC에서 반복 코드 감소
- SQL과 Java 코드 분리
	SQL은 Mapper.xml에 작성하고 자바는 메소드 호출 중심으로 작성
- 객체 매핑 편하다
- 협업에 유리하다

4) MyBatis 기술
	DBCP : DB 연결을 효율적으로 관리하는 기술 
		커넥션을 미리 만들어두고 빌려줌
	JNDI : DBCP 설정을 서버에 등록하고 이름으로 찾아 쓰게 해주는 기술
		커넥션 풀 설정을 이름으로 찾아오게 해줌
	MyBatis : JDBC를 더 편하게 쓰도록 도와주는 DB 프레임워크
		SQL 실행을 훨씬 편하게 해줌

2. DBCP(Database Connection Pool)
	ex) 사용자가 요청할 때마다 매번 DB서버 연결 -> 아이디/비밀번호 인증 -> 세션 만들고 -> 작업 끝나면 끊김
		반복하면 비용이 크다
		사용자 100명이 동시에 로그인하거나 게시글 작성하거나 목록을 보면 매번 새 연결을 만들고 끊는 방식은 비효율적

1) DBCP 역할
	Connection 객체를 모아놓은 풀(Pool)
	미리 일정 개수의 커넥션을 생성해놓고 보관해둔다.
	필요할 때마다 꺼내쓰고 다 쓰면 다시 반납한다.
	
	요청 -> pool에서 Connection 하나 꺼냄 -> SQL 실행 -> 사용 끝남 -> pool에 반납

2) 장점
	DB 연결 생성 비용 절감
	응답 속도 향상
	많은 요청을 안정적으로 처리
	자원 관리가 쉬워짐

3. JNDI(Java Naming and Directory Interface)
	자바 외부에서 외부 자원(Resource)을 이름으로 찾아서 사용할 수 있게 해주는 기술
	외부 자원 : DB연결정보, DBCP, 메일 서버 설정, 환경 설정 값
	=> 자바 코드 안에 DB 정보를 하드코딩 하지 않고 서버에 등록된 자원을 이름으로 찾아서 쓰는 방식

1) JNDI 필요한 이유
	DB 연결 설정을 코드에 직접 작성하면
	URL, 계정, 비밀번호가 코드에 박힘 / 운영환경이 바뀌면 소스코드 수정해야함 / 보안상 취약함 / 여러 프로젝트에서 재사용 어려움
	이러한 문제가 있다.

	톰캣 같은 서버에 DataSource를 등록하고 자바에선느 그 이름만 찾아서 사용한다.
	ex) 서버에 jdbc/my 이름으로  db자원 등록 -> 자바 코드에서는 jdbc/my를 lookup해서 사용

2) DBCP와 JNDI를 같이 쓰는 이유
	DBCP : 커넥션 풀을 관리하는 기술
	JNDI : DBCP에 만들ㅇ저ㅣㄴ DataSource를 이름으로 찾아오게 해주는 기술

	DBCP는 커넥션을 관리하고 JNDI는 그 커넥션 풀을 이름으로 찾아오게한다.

+) DBCP
   커넥션 객체를 이용하여 db에 접근하는데 사용할 때마다 연결과 종료를 반복하면 비효율적이다
   DBCP는 일정량의 커넥션 객체를 미리 만들어놓고 Connection Pool에 보관하여 필요할 때마다 커넥션 객체를 꺼내서
   사용하고 반납하는 방식으로 관리한다

+) JNDI
   외부의 자원을 자바의 객체로 가져와서 사용할 때 필요한 기술이며 API이다
   DBCP와 함께 사용하는 이유는 DBCP가 Connection 객체를 생성할 때 필요한 여러 설정값을 저장하고 관리해주어 
   DBCP의 효율을 높여줄 수 있기 때문이다

4. MyBatis 동작 순서 ★★★★★
	요청이 오기 전에 어플리케이션은 MyBatis 사용 준비를 한다
	(1) SqlSessionFactoryBuilder
		설정 파일을 읽어서 공장을 만드는 빌더 역할
	(2) mybatis-config.xml
		MyBatis 전역 설정 파일
		- db연결정보, 별칭(typeAlias), 매퍼 등록, 환경설정
	(3) SqlSessionFactory
		SqlSession을 만들어내는 공장

	요청 처리 단계(사용자의 요청이 들어오면)
	ex) 로그인, 회원가입, 게시글 목록 조회, 게시글 작성 등
	(4) 요청 발생
	(5) SqlSessionFactory에서 SqlSession 생성
		SqlSession은 DB 작업을 수행하는 핵심 객체
	(6) SqlSession이 Mapper와 연결됨
		개발자가 지정한 SQL 실행
	(7) Mapper.xml 또는 Mapper 인터페이스의 SQL을 수행
		등록된 SQL문을 찾아 실행
	(8) 결과 반환
		조회 객체 / 리스트 현활, 삽입/수정/삭젬녀 성공 행 수 반환
	(9) 필요 시 commit / rollback
		insert/update/delete 트랜잭션 처리 필요
	(10) SqlSession 종료
		사용 끝난 세션 정리

	시작하기 > mybatis-x.x.x.jar 클릭(github 사이트로 이동)

2) config.xml
	MyBatis의 핵심 설정 파일
	DB 연결 방법과 환경 설정
(1) XML 선언
<?xml version="1.0" encoding="UTF-8"?>
- DECODE 선언 : MyBatis 규칙에 맞는지 검사하는 문서 정의(DTD)
<!DOCTYPE configuration PUBLIC "-//mybatis.org//DTD Config 3.0//EN" "http://mybatis.org/dtd/mybatis-3-config.dtd">

- configuration : MyBatis 설정 파일의 최상위 태그
<configuration>
	- environments : 여러 DB 환경을 설정할 수 있는 태그(default="oracle" 기본 DB환경은 oracle을 사용한다)
	<environments default="oracle">
		- enviroment 하나의 DB 환경 설정
		<environment id="oracle">
			- 트랜잭션 관리 방식 설정(트랜잭션 : 여러 SQL 작업을 하나의 작업처럼 처리)
			- type="JDBC" : JDBC 트랜잭션 사용)or type="MANAGED" : 컨테이너 관리
			<transactionManager type="JDBC"></transactionManager>
			- DB 연결 방법 설정
			- type 옵션 POOLED : Connection Pool 사용
				UNPOOLED : 매번 새 연결
				JNDI : 서버에서 가져옴
			<dataSource type="POOLED">
				- DB 설정
				- Oracle JDBC 드라이버
				<property name="driver"
				- URL : DB 접속 구조	
			</dataSource>
		</environment>
	</environments>
</configuration>
